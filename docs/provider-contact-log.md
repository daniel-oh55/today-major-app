# Provider 문의 로그

> **작성 기준:** Phase 16 — 실제 문의 전 준비 단계
>
> **현재 상태:** 모든 업체 `not_contacted` — 아직 문의를 발송하지 않았습니다.
>
> **문의 발송 전 체크리스트:**
> - `docs/commercial-api-checklist.md` 전체 검토
> - `docs/provider-contact-template.md` 영문 문의 템플릿 확인
> - `docs/commercial-api-due-diligence.md` 해당 항목 업데이트 준비
>
> **연관 문서:**
> - Due Diligence: `docs/commercial-api-due-diligence.md`
> - 문의 템플릿: `docs/provider-contact-template.md`
> - POC 선정 기준: `docs/api-poc-selection-criteria.md`
> - 리스크 등록부: `docs/commercial-api-risk-register.md`

## 로그 항목 구조

각 문의 기록은 아래 항목으로 구성됩니다.

| 필드 | 설명 |
|------|------|
| 업체명 | Provider 이름 |
| 문의일 | 문의 발송 날짜 (YYYY-MM-DD) |
| 문의 채널 | email / contact form / live chat / 영업팀 등 |
| 문의 대상 | 담당자명 또는 contact URL |
| 보낸 질문 요약 | `docs/provider-contact-template.md` 기반 항목 중 어떤 항목을 물었는지 |
| 회신 상태 | `not_contacted` / `contacted` / `awaiting_reply` / `replied` / `no_reply` |
| 회신 요약 | 회신 내용 요약 (없으면 "미회신") |
| follow-up 필요 | Y / N |
| 다음 액션 | 다음 단계 (예: 재문의 예정, 계약 검토, 보류) |
| 내부 판단 | 확인 필요 / POC 후보 / 보류 / 제외 |
